# 7. EDA de series de tiempo CAM

## Objetivo de la etapa

Esta etapa corresponde al analisis exploratorio de datos para series de tiempo. Se realiza despues de la extraccion, transformacion, diagnostico temporal e imputacion general.

El objetivo es entender el comportamiento temporal de las variables de las estaciones CAM, especialmente `nivel_m`, antes de pasar a una etapa posterior de modelos.

En esta fase no se entrenan modelos. Solo se generan reportes y graficas para interpretar los datos.

## Lugar dentro del flujo del proyecto

El orden actual del proyecto queda asi:

```text
1. Extraccion de datos
2. Transformacion y limpieza
3. Analisis de calidad temporal
4. Imputacion especifica de nivel_m
5. Interpretacion de resultados de imputacion de nivel_m
6. Estado del proyecto e imputacion general
7. EDA de series de tiempo
```

Esta etapa usa como entrada la base integrada generada en la etapa 6.

## Archivo de entrada

El EDA usa:

```text
ETL_CAM/transform/data/imputacion_general/variables_modelo_imputadas.csv
```

Este archivo se considera la base oficial integrada porque contiene las variables principales, las banderas de imputacion y las columnas de calidad.

## Reportes generados

### resumen_eda_general.csv

Resume el estado general de la base usada para el EDA:

```text
registros
estaciones
fecha_minima
fecha_maxima
variables_modelo
```

Este archivo sirve para mostrar el tamano de la base y el rango temporal analizado.

### resumen_eda_estaciones.csv

Resume informacion por estacion:

```text
registros
fecha_minima
fecha_maxima
nivel_m_promedio
nivel_m_minimo
nivel_m_maximo
nivel_m_desviacion
```

Este reporte permite identificar que estaciones tienen mayor o menor nivel de agua y cuales presentan mas variabilidad.

### resumen_variables.csv

Resume el comportamiento de cada variable por estacion:

```text
disponibles
faltantes
disponible_pct
media
minimo
maximo
desviacion
imputados
imputados_pct
calidad_real
calidad_alta
calidad_media
calidad_baja
calidad_sin_dato
```

Este archivo es clave para interpretar la calidad de cada variable antes de usarla en correlaciones o modelos.

### eventos_nivel_extremo.csv

Identifica registros donde `nivel_m` esta por encima del percentil 95 de cada estacion.

Esto no significa automaticamente inundacion, pero sirve para detectar posibles momentos de nivel alto que deben revisarse.

### correlacion_variables.csv

Contiene la matriz de correlacion entre variables numericas.

Esta matriz ayuda a revisar relaciones generales entre variables como:

```text
nivel_m
lluvia_mm
velocidad_agua_m_s
temperatura_c
humedad_pct
presion_atmosferica_hpa
```

## Graficas generadas

### nivel_m_por_estacion.svg

Muestra el nivel del agua promedio diario por estacion. Sirve para ver tendencias, cambios de largo plazo y diferencias generales entre estaciones.

### nivel_m_promedio_mensual.svg

Muestra el promedio mensual de `nivel_m`. Ayuda a identificar posibles patrones estacionales.

### nivel_m_promedio_horario.svg

Muestra el promedio horario de `nivel_m`. Permite revisar si existe algun patron dentro del dia.

### boxplot_nivel_por_estacion.svg

Compara la distribucion de `nivel_m` entre estaciones. Sirve para identificar estaciones con niveles mas altos, mayor dispersion o comportamientos atipicos.

### correlacion_variables.svg

Muestra el mapa de correlacion entre variables. Debe leerse con cuidado porque algunas variables fueron imputadas.

### lluvia_vs_nivel.svg

Compara lluvia diaria acumulada contra nivel promedio diario. Sirve para revisar si los aumentos de lluvia se relacionan con aumentos en el nivel del agua.

### calidad_imputacion_por_variable.svg

Muestra que porcentaje de cada variable fue imputado. Esta grafica es importante para saber que variables son mas confiables y cuales dependen mucho de la imputacion.

## Importancia de la imputacion en el EDA

El archivo usado ya tiene imputacion general. Por eso el EDA no solo mira los valores finales, sino tambien las columnas de calidad e imputacion.

Esto es importante porque un valor real no tiene la misma confianza que un valor estimado.

Por eso, al interpretar resultados, se debe revisar:

```text
variable_fue_imputado
variable_calidad
```

Ejemplo:

```text
nivel_m_fue_imputado
nivel_m_calidad
lluvia_mm_fue_imputado
lluvia_mm_calidad
```
