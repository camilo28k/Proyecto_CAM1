# 7. EDA de series de tiempo CAM

## Objetivo de la etapa

Esta etapa corresponde al análisis exploratorio de datos para series de tiempo. Se realiza después de la extracción, transformación, diagnóstico temporal e imputación general.

El objetivo es entender el comportamiento temporal de las variables de las estaciones CAM, especialmente `nivel_m`, antes de pasar a una etapa posterior de modelos.

En esta fase no se entrenan modelos. Solo se generan reportes y gráficas para interpretar los datos.

## Lugar dentro del flujo del proyecto

El orden actual del proyecto queda así:

```text
1. Extracción de datos
2. Transformación y limpieza
3. Análisis de calidad temporal
4. Imputación específica de nivel_m
5. Interpretación de resultados de imputación de nivel_m
6. Estado del proyecto e imputación general
7. EDA de series de tiempo
```

Esta etapa usa como entrada la base integrada generada en la etapa 6.

## Archivo de entrada

El EDA usa:

```text
ETL_CAM/transform/data/imputacion_general/variables_modelo_imputadas.csv
```

Este archivo se considera la base oficial integrada porque contiene las variables principales, las banderas de imputación y las columnas de calidad.

## Reportes generados

### resumen_eda_general.csv

Resume el estado general de la base usada para el EDA:

```text
registros
estaciones
fecha_minima
fecha_maxima
variables_modelo
```

Este archivo sirve para mostrar el tamaño de la base y el rango temporal analizado.

### resumen_eda_estaciones.csv

Resume información por estación:

```text
registros
fecha_minima
fecha_maxima
nivel_m_promedio
nivel_m_minimo
nivel_m_maximo
nivel_m_desviacion
```

Este reporte permite identificar qué estaciones tienen mayor o menor nivel de agua y cuáles presentan más variabilidad.

### resumen_variables.csv

Resume el comportamiento de cada variable por estación:

```text
disponibles
faltantes
disponible_pct
media
minimo
maximo
desviacion
imputados
imputados_pct
calidad_real
calidad_alta
calidad_media
calidad_baja
calidad_sin_dato
```

Este archivo es clave para interpretar la calidad de cada variable antes de usarla en correlaciones o modelos.

### eventos_nivel_extremo.csv

Identifica registros donde `nivel_m` está por encima del percentil 95 de cada estación.

Esto no significa automáticamente inundación, pero sirve para detectar posibles momentos de nivel alto que deben revisarse.

### correlacion_variables.csv

Contiene la matriz de correlación entre variables numéricas.

Esta matriz ayuda a revisar relaciones generales entre variables como:

```text
nivel_m
lluvia_mm
velocidad_agua_m_s
temperatura_c
humedad_pct
presion_atmosferica_hpa
```

## Gráficas generadas

### nivel_m_por_estacion.svg

Muestra el nivel del agua promedio diario por estación. Sirve para ver tendencias, cambios de largo plazo y diferencias generales entre estaciones.

### nivel_m_promedio_mensual.svg

Muestra el promedio mensual de `nivel_m`. Ayuda a identificar posibles patrones estacionales.

### nivel_m_promedio_horario.svg

Muestra el promedio horario de `nivel_m`. Permite revisar si existe algún patrón dentro del día.

### boxplot_nivel_por_estacion.svg

Compara la distribución de `nivel_m` entre estaciones. Sirve para identificar estaciones con niveles más altos, mayor dispersión o comportamientos atípicos.

### correlacion_variables.svg

Muestra el mapa de correlación entre variables. Debe leerse con cuidado porque algunas variables fueron imputadas.

### lluvia_vs_nivel.svg

Compara lluvia diaria acumulada contra nivel promedio diario. Sirve para revisar si los aumentos de lluvia se relacionan con aumentos en el nivel del agua.

### calidad_imputacion_por_variable.svg

Muestra qué porcentaje de cada variable fue imputado. Esta gráfica es importante para saber qué variables son más confiables y cuáles dependen mucho de la imputación.

## Importancia de la imputación en el EDA

El archivo usado ya tiene imputación general. Por eso el EDA no solo mira los valores finales, sino también las columnas de calidad e imputación.

Esto es importante porque un valor real no tiene la misma confianza que un valor estimado.

Por eso, al interpretar resultados, se debe revisar:

```text
variable_fue_imputado
variable_calidad
```

Ejemplo:

```text
nivel_m_fue_imputado
nivel_m_calidad
lluvia_mm_fue_imputado
lluvia_mm_calidad
```
