# Documento del proceso de transformacion CAM

## Objetivo

El proceso de transformacion tiene como objetivo limpiar, normalizar y estructurar la informacion extraida de las estaciones CAM para dejarla lista para analisis, carga o etapas posteriores del flujo ETL.

Esta etapa trabaja sobre el archivo consolidado generado por el extractor y produce una version mas ordenada, tipada y validada de la serie de tiempo.

## Ubicacion del codigo

El codigo del transformador se encuentra en:

```text
ETL_CAM/transform/transformador.py
```

## Entrada del proceso

El archivo de entrada es:

```text
ETL_CAM/extract/data/cam_extraido.csv
```

Este archivo contiene la informacion consolidada de todas las estaciones y todos los anos procesados durante la extraccion.

## Salida del proceso

El transformador genera el archivo:

```text
ETL_CAM/transform/data/cam_transformado.csv
```

Tambien genera un archivo de log:

```text
ETL_CAM/transform/transformador_cam.log
```

## Descripcion general del proceso

El transformador toma la informacion extraida y aplica reglas de limpieza y normalizacion. Su funcion principal es dejar una base coherente para trabajar con series de tiempo por estacion.

El proceso no realiza imputacion de datos faltantes. Los valores ausentes se conservan como nulos para no alterar artificialmente la serie temporal. La imputacion se realiza mas adelante.

## Pasos realizados

1. Carga el archivo `cam_extraido.csv`.
2. Normaliza los nombres de columnas.
3. Convierte la columna de fecha a tipo fecha-hora.
4. Limpia columnas de texto como `estacion` y `archivo_origen`.
5. Une columnas equivalentes que representan la misma variable.
6. Convierte las variables de medicion a formato numerico.
7. Aplica validaciones basicas de rangos fisicos.
8. Elimina registros sin fecha o sin estacion.
9. Ordena los datos por `estacion` y `fecha_hora`.
10. Combina registros duplicados por estacion y fecha.
11. Crea columnas de calidad del registro.
12. Guarda el resultado transformado en `cam_transformado.csv`.

## Normalizacion de columnas

El proceso estandariza los nombres de columnas para evitar problemas causados por:

- Espacios.
- Mayusculas y minusculas.
- Parentesis.
- Simbolos como grados.
- Separadores inconsistentes.

Por ejemplo, columnas como:

```text
temperatura_(C)
radiacion_solar_(W/m2)
velocidad_viento_(m/s)
```

se normalizan a nombres mas consistentes para el procesamiento.

## Unificacion de columnas equivalentes

Algunos archivos contienen la misma variable con nombres diferentes. El transformador combina estas columnas para evitar duplicidad.

Las principales unificaciones son:

```text
humedad_% + humedad_relativa_% -> humedad_pct
presion_atmosferica_hpa + pres_atmosferica_hpa -> presion_atmosferica_hpa
radiacion_solar_w_m2 + rad_u_w_m2 -> radiacion_solar_w_m2
direccion_viento_ + direccion_viento_m_s -> direccion_viento_grados
```

Cuando existen varias columnas candidatas, se toma el primer valor disponible por fila.

## Variables de medicion conservadas

El archivo transformado conserva las siguientes variables principales:

```text
nivel_m
temperatura_c
humedad_pct
radiacion_solar_w_m2
lluvia_mm
direccion_viento_grados
velocidad_viento_m_s
presion_atmosferica_hpa
voltaje_bateria_vdc
velocidad_agua_m_s
```

Tambien conserva columnas de trazabilidad:

```text
fecha_hora
estacion
anio
archivo_origen
```

## Conversion de tipos de datos

La columna `fecha_hora` se convierte a tipo fecha-hora usando interpretacion de dia primero, debido al formato comun de los datos originales.

Las variables de medicion se convierten a numericas. Si algun valor no puede convertirse correctamente, queda como nulo.

## Validacion de rangos

El transformador aplica rangos fisicos basicos para detectar valores claramente invalidos. Algunos ejemplos:

```text
temperatura_c: -10 a 60
humedad_pct: 0 a 100
radiacion_solar_w_m2: 0 a 1600
lluvia_mm: 0 a 500
direccion_viento_grados: 0 a 360
velocidad_viento_m_s: 0 a 80
presion_atmosferica_hpa: 500 a 1100
voltaje_bateria_vdc: 0 a 30
nivel_m: -10 a 100
velocidad_agua_m_s: 0 a 20
```

Si un valor esta fuera del rango esperado, se marca como nulo. No se reemplaza por otro valor.

## Manejo de duplicados

Como se trabaja con series de tiempo, el identificador natural de un registro es:

```text
estacion + fecha_hora
```

Si existen registros duplicados para la misma estacion y fecha, el transformador los combina y conserva el primer valor disponible por columna. Esto evita perder informacion cuando una fila tiene datos faltantes pero otra fila duplicada tiene una medicion disponible.

## Columnas de calidad

El transformador crea columnas adicionales para evaluar la calidad de cada registro:

### fecha_valida

Indica si el registro tiene una fecha correctamente interpretada.

### tiene_medicion

Indica si el registro tiene al menos una variable de medicion disponible.

### registro_valido

Indica si el registro tiene estacion, fecha valida y al menos una medicion.

### cantidad_variables_disponibles

Cuenta cuantas variables de medicion tienen dato en la fila.

### cantidad_variables_faltantes

Cuenta cuantas variables de medicion estan vacias en la fila.


## Metricas registradas

Al finalizar, el transformador informa:

```text
Registros de entrada
Registros de salida
Duplicados eliminados o combinados
Tiempo de ejecucion
Archivo generado
```

## Comando de ejecucion

Desde la raiz del proyecto:

```powershell
python ETL_CAM\transform\transformador.py
```
