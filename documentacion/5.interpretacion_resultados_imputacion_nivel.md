Este resultado debe entenderse como una revisión especializada de `nivel_m`, no como la tabla final única del proyecto. Para correlaciones y modelos, la salida oficial integrada se genera con `imputacion_general.py`.

## Archivo nivel_m_imputado.csv

Este archivo contiene la serie temporal de `nivel_m` después del proceso de imputación.

Su función principal es permitir la interpretación de la calidad de la variable objetivo. La base final para modelos debe tomar la estructura integrada de:

```text
ETL_CAM/transform/data/imputacion_general/variables_modelo_imputadas.csv
```

Las columnas principales son:

```text
fecha_hora
estacion
nivel_m_original
nivel_m_imputado
fue_imputado_nivel
metodo_imputacion_nivel
calidad_imputacion_nivel
```

## Significado de las columnas

### fecha_hora

Fecha y hora del registro en la serie temporal. El proceso trabaja con una frecuencia de 5 minutos.

Algunos registros pueden aparecer con fecha `2026-01-01 00:00:00`. Esto ocurre porque datos cercanos al final del año, por ejemplo `2025-12-31 23:58`, fueron redondeados a la grilla de 5 minutos. No significa que exista una nueva campaña de datos de 2026; es un efecto del redondeo temporal.

### estacion

Nombre de la estación CAM a la que pertenece el registro.

### nivel_m_original

Valor real registrado por la estación. Si esta columna está vacía, significa que no había medición original para ese instante.

### nivel_m_imputado

Valor final después del proceso. Puede corresponder a:

- Un valor real observado.
- Un valor interpolado.
- Un valor estimado con promedio histórico.

Esta es la columna que se usaría como serie completa para análisis posteriores, siempre teniendo en cuenta la columna de calidad.

### fue_imputado_nivel

Indica si el valor fue calculado por el proceso de imputación.

```text
False: el valor viene de una medición real.
True: el valor fue estimado.
```

### metodo_imputacion_nivel

Indica cómo se obtuvo el valor final.

Los métodos posibles son:

```text
observado
interpolacion_temporal_60min
promedio_historico_mes_dia_hora_minuto
promedio_historico_mes_hora_minuto
sin_imputar
```

### calidad_imputacion_nivel

Indica el nivel de confianza del valor.

```text
real: dato medido por la estación.
alta: dato interpolado en un hueco corto.
media: dato estimado con histórico calendario.
baja: dato estimado con histórico mensual-horario.
sin_dato: no fue posible imputar.
```

## Qué significa que no hayan quedado datos sin imputar

El resumen muestra `0` registros sin imputar en las estaciones procesadas. Esto significa que todos los faltantes pudieron rellenarse con alguna de las reglas definidas:

1. Interpolación temporal.
2. Promedio histórico calendario.
3. Promedio histórico mensual-horario.

Aunque esto es positivo para tener una serie completa, también implica que una parte importante de los datos no son mediciones reales. Por eso es fundamental conservar las columnas de método y calidad.

## Recomendación para el siguiente paso

El siguiente paso no debería ser entrenar el modelo directamente con este archivo aislado. Primero se recomienda usar la imputación general para crear una base integrada con `nivel_m` y las variables predictoras.

Después de generar la base integrada, se recomienda hacer una revisión visual y estadística de la serie imputada.

Se recomienda generar gráficas por estación comparando:

```text
nivel_m_original
nivel_m_imputado
metodo_imputacion_nivel
```

Especialmente se deben revisar:

- Picos máximos de nivel.
- Cambios bruscos.
- Períodos con mucha imputación histórica.
- Diferencias entre datos reales e imputados.
- Estaciones con baja proporción de datos reales.

## Conclusión

El proceso de imputación logró construir series completas de `nivel_m` para seis estaciones. Sin embargo, no todos los valores tienen la misma confiabilidad. Para análisis de inundaciones, se debe dar prioridad a los valores reales e interpolados, y usar con precaución los valores estimados por promedios históricos, especialmente los de calidad baja.

El resultado es útil como base para análisis y modelado, siempre que se mantenga la trazabilidad entre dato real e imputado.
