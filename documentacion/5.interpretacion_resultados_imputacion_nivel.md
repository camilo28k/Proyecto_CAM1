# Interpretacion de resultados de imputacion de nivel CAM

Este documento explica el significado de los archivos generados por el proceso `imputacion_nivel.py`, especialmente el resultado de la imputacion de la variable `nivel_m`, usada como variable principal para el analisis de posibles inundaciones.

Este resultado debe entenderse como una revision especializada de `nivel_m`, no como la tabla final unica del proyecto. Para correlaciones y modelos, la salida oficial integrada se genera con `imputacion_general.py`.

## Archivo nivel_m_imputado.csv

Este archivo contiene la serie temporal de `nivel_m` despues del proceso de imputacion.

Su funcion principal es permitir la interpretacion de la calidad de la variable objetivo. La base final para modelos debe tomar la estructura integrada de:

```text
ETL_CAM/transform/data/imputacion_general/variables_modelo_imputadas.csv
```

Las columnas principales son:

```text
fecha_hora
estacion
nivel_m_original
nivel_m_imputado
fue_imputado_nivel
metodo_imputacion_nivel
calidad_imputacion_nivel
```

## Significado de las columnas

### fecha_hora

Fecha y hora del registro en la serie temporal. El proceso trabaja con una frecuencia de 5 minutos.

Algunos registros pueden aparecer con fecha `2026-01-01 00:00:00`. Esto ocurre porque datos cercanos al final del ano, por ejemplo `2025-12-31 23:58`, fueron redondeados a la grilla de 5 minutos. No significa que exista una nueva campana de datos de 2026; es un efecto del redondeo temporal.

### estacion

Nombre de la estacion CAM a la que pertenece el registro.

### nivel_m_original

Valor real registrado por la estacion. Si esta columna esta vacia, significa que no habia medicion original para ese instante.

### nivel_m_imputado

Valor final despues del proceso. Puede corresponder a:

- Un valor real observado.
- Un valor interpolado.
- Un valor estimado con promedio historico.

Esta es la columna que se usaria como serie completa para analisis posteriores, siempre teniendo en cuenta la columna de calidad.

### fue_imputado_nivel

Indica si el valor fue calculado por el proceso de imputacion.

```text
False: el valor viene de una medicion real.
True: el valor fue estimado.
```

### metodo_imputacion_nivel

Indica como se obtuvo el valor final.

Los metodos posibles son:

```text
observado
interpolacion_temporal_60min
promedio_historico_mes_dia_hora_minuto
promedio_historico_mes_hora_minuto
sin_imputar
```

### calidad_imputacion_nivel

Indica el nivel de confianza del valor.

```text
real: dato medido por la estacion.
alta: dato interpolado en un hueco corto.
media: dato estimado con historico calendario.
baja: dato estimado con historico mensual-horario.
sin_dato: no fue posible imputar.
```

## Que significa que no hayan quedado datos sin imputar

El resumen muestra `0` registros sin imputar en las estaciones procesadas. Esto significa que todos los faltantes pudieron rellenarse con alguna de las reglas definidas:

1. Interpolacion temporal.
2. Promedio historico calendario.
3. Promedio historico mensual-horario.

Aunque esto es positivo para tener una serie completa, tambien implica que una parte importante de los datos no son mediciones reales. Por eso es fundamental conservar las columnas de metodo y calidad.

## Recomendacion para el siguiente paso

El siguiente paso no deberia ser entrenar el modelo directamente con este archivo aislado. Primero se recomienda usar la imputacion general para crear una base integrada con `nivel_m` y las variables predictoras.

Despues de generar la base integrada, se recomienda hacer una revision visual y estadistica de la serie imputada.

Se recomienda generar graficas por estacion comparando:

```text
nivel_m_original
nivel_m_imputado
metodo_imputacion_nivel
```

Especialmente se deben revisar:

- Picos maximos de nivel.
- Cambios bruscos.
- Periodos con mucha imputacion historica.
- Diferencias entre datos reales e imputados.
- Estaciones con baja proporcion de datos reales.

## Conclusion

El proceso de imputacion logro construir series completas de `nivel_m` para seis estaciones. Sin embargo, no todos los valores tienen la misma confiabilidad. Para analisis de inundaciones, se debe dar prioridad a los valores reales e interpolados, y usar con precaucion los valores estimados por promedios historicos, especialmente los de calidad baja.

El resultado es util como base para analisis y modelado, siempre que se mantenga la trazabilidad entre dato real e imputado.
