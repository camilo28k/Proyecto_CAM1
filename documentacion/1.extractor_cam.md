# Documento del proceso de extraccion CAM

## Objetivo

El proceso de extraccion tiene como objetivo recopilar la informacion historica de las estaciones CAM almacenada en la carpeta `Datos_CAM` y consolidarla en un unico archivo tabular. Esta etapa corresponde al primer paso del flujo ETL: obtener los datos originales desde sus archivos fuente sin aplicar transformaciones profundas ni imputaciones.

## Ubicacion del codigo

El codigo del extractor se encuentra en:

```text
ETL_CAM/extract/extractor.py
```

## Entrada del proceso

La entrada principal es la carpeta:

```text
Datos_CAM
```

Dentro de esta carpeta se encuentran subcarpetas por estacion, por ejemplo:

```text
Datos_CAM/El_Batan
Datos_CAM/El_Meson
Datos_CAM/Las_Juntas
Datos_CAM/La_Lindosa
Datos_CAM/Los_Micos
Datos_CAM/Puente_Bruselas
Datos_CAM/Santa_Rosalia
Datos_CAM/San_Bartolo
Datos_CAM/San_Benito
```

Cada subcarpeta contiene archivos historicos por ano en formato `.csv`, `.xlsx` o `.xls`. Los archivos PDF de observaciones no se procesan en esta etapa porque no contienen datos tabulares principales para el ETL.

## Salida del proceso

El extractor genera el archivo:

```text
ETL_CAM/extract/data/cam_extraido.csv
```

Tambien genera un archivo de log con el detalle de ejecucion:

```text
ETL_CAM/extract/extractor_cam.log
```

## Descripcion general del proceso

El extractor recorre automaticamente todas las carpetas de estaciones ubicadas dentro de `Datos_CAM`. Para cada estacion, revisa los archivos disponibles y procesa solamente aquellos que tienen extension `.csv`, `.xlsx` o `.xls`.

Cuando encuentra un archivo CSV, lo lee usando separador `;` y decimal `,`, ya que los datos originales usan formato latino para los numeros decimales. Cuando encuentra un archivo Excel, lo lee con `pandas` usando el motor `openpyxl`.

Una vez leido cada archivo, el extractor agrega columnas de contexto para conservar el origen de los datos:

```text
estacion
anio
archivo_origen
```

Estas columnas permiten saber de que estacion, ano y archivo proviene cada registro.

## Pasos realizados

1. Verifica que exista la carpeta `Datos_CAM`.
2. Recorre cada subcarpeta de estacion.
3. Identifica archivos `.csv`, `.xlsx` y `.xls`.
4. Lee los archivos CSV con separador `;` y decimal `,`.
5. Lee los archivos Excel mediante `openpyxl`.
6. Normaliza parcialmente los nombres de columnas para reducir problemas con espacios, parentesis y simbolos.
7. Extrae el ano desde el nombre del archivo.
8. Agrega las columnas `estacion`, `anio` y `archivo_origen`.
9. Une todos los archivos procesados en un solo DataFrame.
10. Guarda la informacion consolidada en `cam_extraido.csv`.
11. Registra metricas de ejecucion en el log.

## Columnas agregadas por el extractor

### estacion

Indica la estacion de origen del registro. Se obtiene a partir del nombre de la carpeta donde estaba almacenado el archivo.

### anio

Indica el ano asociado al archivo. Se extrae desde el nombre del archivo mediante una expresion regular que busca anos como `2017`, `2020`, `2025`, entre otros.

### archivo_origen

Guarda el nombre del archivo original. Esto permite trazabilidad para saber exactamente de donde provino cada fila.

## Manejo de errores

Si un archivo no puede leerse, el extractor no detiene todo el proceso. En cambio:

- Registra el error en el log.
- Incrementa el contador de archivos fallidos.
- Continua procesando los demas archivos.

Esto permite obtener la mayor cantidad posible de datos aunque algun archivo tenga problemas.

## Metricas registradas

Al finalizar, el extractor informa:

```text
Registros extraidos
Archivos procesados
Archivos fallidos
Tiempo de ejecucion
Archivo generado
```

Estas metricas ayudan a validar si la extraccion fue completa o si hubo archivos que requieren revision.

## Consideraciones importantes

El extractor no realiza imputacion de datos faltantes. Tampoco decide si un valor ausente debe reemplazarse por cero, promedio, interpolacion u otro metodo. Su responsabilidad es conservar la informacion original y dejarla consolidada para la siguiente etapa.

La limpieza profunda, unificacion de columnas y validacion de rangos se realiza posteriormente en el transformador.

## Dependencias

El proceso requiere:

```text
pandas
openpyxl
```

Si los archivos Excel fallan con un mensaje relacionado con `openpyxl`, se debe instalar la dependencia:

```powershell
pip install openpyxl
```

## Comando de ejecucion

Desde la raiz del proyecto:

```powershell
python ETL_CAM\extract\extractor.py
```
