# Documento del proceso de extracción CAM

## Objetivo

El proceso de extracción tiene como objetivo recopilar la información histórica de las estaciones CAM almacenada en la carpeta `Datos_CAM` y consolidarla en un único archivo tabular. Esta etapa corresponde al primer paso del flujo ETL: obtener los datos originales desde sus archivos fuente sin aplicar transformaciones profundas ni imputaciones.

## Ubicación del código

El código del extractor se encuentra en:

```text
ETL_CAM/extract/extractor.py
```

## Entrada del proceso

La entrada principal es la carpeta:

```text
Datos_CAM
```

Dentro de esta carpeta se encuentran subcarpetas por estación, por ejemplo:

```text
Datos_CAM/El_Batan
Datos_CAM/El_Meson
Datos_CAM/Las_Juntas
Datos_CAM/La_Lindosa
Datos_CAM/Los_Micos
Datos_CAM/Puente_Bruselas
Datos_CAM/Santa_Rosalia
Datos_CAM/San_Bartolo
Datos_CAM/San_Benito
```

Cada subcarpeta contiene archivos históricos por año en formato `.csv`, `.xlsx` o `.xls`. Los archivos PDF de observaciones no se procesan en esta etapa porque no contienen datos tabulares principales para el ETL.

## Salida del proceso

El extractor genera el archivo:

```text
ETL_CAM/extract/data/cam_extraido.csv
```

También genera un archivo de log con el detalle de ejecución:

```text
ETL_CAM/extract/extractor_cam.log
```

## Descripción general del proceso

El extractor recorre automáticamente todas las carpetas de estaciones ubicadas dentro de `Datos_CAM`. Para cada estación, revisa los archivos disponibles y procesa solamente aquellos que tienen extensión `.csv`, `.xlsx` o `.xls`.

Cuando encuentra un archivo CSV, lo lee usando separador `;` y decimal `,`, ya que los datos originales usan formato latino para los números decimales. Cuando encuentra un archivo Excel, lo lee con `pandas` usando el motor `openpyxl`.

Una vez leído cada archivo, el extractor agrega columnas de contexto para conservar el origen de los datos:

```text
estacion
anio
archivo_origen
```

Estas columnas permiten saber de qué estación, año y archivo proviene cada registro.

## Pasos realizados

1. Verifica que exista la carpeta `Datos_CAM`.
2. Recorre cada subcarpeta de estación.
3. Identifica archivos `.csv`, `.xlsx` y `.xls`.
4. Lee los archivos CSV con separador `;` y decimal `,`.
5. Lee los archivos Excel mediante `openpyxl`.
6. Normaliza parcialmente los nombres de columnas para reducir problemas con espacios, paréntesis y símbolos.
7. Extrae el año desde el nombre del archivo.
8. Agrega las columnas `estacion`, `anio` y `archivo_origen`.
9. Une todos los archivos procesados en un solo DataFrame.
10. Guarda la información consolidada en `cam_extraido.csv`.
11. Registra métricas de ejecución en el log.

## Columnas agregadas por el extractor

### estacion

Indica la estación de origen del registro. Se obtiene a partir del nombre de la carpeta donde estaba almacenado el archivo.

### anio

Indica el año asociado al archivo. Se extrae desde el nombre del archivo mediante una expresión regular que busca años como `2017`, `2020`, `2025`, entre otros.

### archivo_origen

Guarda el nombre del archivo original. Esto permite trazabilidad para saber exactamente de dónde provino cada fila.

## Manejo de errores

Si un archivo no puede leerse, el extractor no detiene todo el proceso. En cambio:

- Registra el error en el log.
- Incrementa el contador de archivos fallidos.
- Continúa procesando los demás archivos.

Esto permite obtener la mayor cantidad posible de datos aunque algún archivo tenga problemas.

## Métricas registradas

Al finalizar, el extractor informa:

```text
Registros extraídos
Archivos procesados
Archivos fallidos
Tiempo de ejecución
Archivo generado
```

Estas métricas ayudan a validar si la extracción fue completa o si hubo archivos que requieren revisión.

## Consideraciones importantes

El extractor no realiza imputación de datos faltantes. Tampoco decide si un valor ausente debe reemplazarse por cero, promedio, interpolación u otro método. Su responsabilidad es conservar la información original y dejarla consolidada para la siguiente etapa.

La limpieza profunda, unificación de columnas y validación de rangos se realiza posteriormente en el transformador.

## Dependencias

El proceso requiere:

```text
pandas
openpyxl
```

Si los archivos Excel fallan con un mensaje relacionado con `openpyxl`, se debe instalar la dependencia:

```powershell
pip install openpyxl
```

## Comando de ejecución

Desde la raíz del proyecto:

```powershell
python ETL_CAM\extract\extractor.py
```
