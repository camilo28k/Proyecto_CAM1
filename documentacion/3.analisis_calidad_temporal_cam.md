# Documento del análisis de calidad temporal CAM

## Objetivo

El análisis de calidad temporal tiene como objetivo revisar la continuidad de las series de tiempo de las estaciones CAM antes de aplicar cualquier método de imputación o interpolación.

Esta etapa es especialmente importante porque la variable principal del proyecto es `nivel_m`, usada para estudiar y predecir posibles inundaciones. En este contexto, rellenar datos faltantes sin conocer primero la estructura temporal puede alterar los picos de nivel del agua y afectar el análisis.

## Ubicación del código

El código se encuentra en:

```text
ETL_CAM/transform/analisis_calidad_temporal.py
```

## Entrada del proceso

El archivo de entrada es:

```text
ETL_CAM/transform/data/cam_transformado.csv
```

Este archivo debe haber sido generado previamente por el transformador.

## Salidas del proceso

El proceso genera dos archivos principales:

```text
ETL_CAM/transform/data/calidad_temporal/resumen_calidad_temporal.csv
ETL_CAM/transform/data/calidad_temporal/huecos_temporales.csv
```

También genera un archivo de log:

```text
ETL_CAM/transform/analisis_calidad_temporal.log
```

## Qué revisa este análisis

El script revisa cada estación de forma independiente. Para cada una calcula:

- Fecha mínima disponible.
- Fecha máxima disponible.
- Frecuencia temporal detectada.
- Cantidad de registros reales.
- Cantidad estimada de registros esperados.
- Cantidad estimada de registros faltantes.
- Porcentaje de cobertura temporal.
- Cantidad de huecos detectados.
- Hueco máximo en minutos.
- Disponibilidad de la variable `nivel_m`.
- Porcentaje de datos disponibles para `nivel_m`.

## Archivo resumen_calidad_temporal.csv

Este archivo contiene una fila por estación. Sus columnas principales son:

```text
estacion
fecha_minima
fecha_maxima
frecuencia_detectada
frecuencia_minutos
registros_reales
registros_esperados
registros_faltantes_estimados
porcentaje_cobertura_temporal
cantidad_huecos_detectados
hueco_maximo_minutos
nivel_m_disponibles
nivel_m_faltantes
nivel_m_disponible_pct
apto_interpolacion_directa
```

## Archivo huecos_temporales.csv

Este archivo contiene el detalle de los saltos o huecos detectados en la serie temporal. Sus columnas principales son:

```text
estacion
inicio_hueco
fin_hueco
duracion_minutos
registros_faltantes_estimados
clasificacion_hueco
```

La clasificación del hueco se define así:

```text
corto: hasta 60 minutos
medio: mayor a 60 minutos y hasta 24 horas
largo: mayor a 24 horas
```

## Por qué se hace antes de imputar

La imputación en series de tiempo no debe hacerse directamente después de limpiar los datos. Primero se debe saber si los faltantes son pequeños, medianos o grandes.

Para la variable `nivel_m`, esto es aún más importante porque los picos del nivel del agua pueden indicar riesgo de inundación. Si se rellenan datos usando promedios históricos sin revisar los huecos, se puede suavizar un evento extremo o crear un comportamiento que no ocurrió realmente.

## Relación con interpolación e imputación

Este análisis no imputa datos. Su función es preparar la decisión.

Con los resultados se puede definir una regla posterior como:

```text
Huecos cortos: interpolación temporal.
Huecos medianos: evaluar promedio histórico por estación, mes, día y hora.
Huecos largos: no imputar automáticamente o marcar como baja confianza.
```

También permite evaluar si la idea de usar datos del año anterior es viable. Por ejemplo, si falta una fecha específica, podría compararse con el comportamiento histórico de la misma estación en fechas similares. Sin embargo, esto debe hacerse con precaución porque los niveles de agua dependen de eventos reales de lluvia y condiciones hidrológicas del momento.

## Variable principal

La variable prioritaria del análisis es:

```text
nivel_m
```

Esto significa que aunque el archivo tenga otras variables meteorológicas o hidrológicas, el diagnóstico pone especial atención en la disponibilidad de nivel del agua.

## Comando de ejecución

Desde la raíz del proyecto:

```powershell
python ETL_CAM\transform\analisis_calidad_temporal.py
```

## Interpretación recomendada

Si una estación tiene alta cobertura temporal y huecos cortos, puede ser buena candidata para interpolación directa.

Si una estación tiene huecos medianos, se debe revisar si conviene usar promedios históricos, ventanas temporales o información de variables auxiliares como lluvia.

Si una estación tiene huecos largos o bajo porcentaje de `nivel_m`, no se recomienda imputar automáticamente para modelar inundaciones sin una justificación adicional.
