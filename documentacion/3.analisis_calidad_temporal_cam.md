# Documento del analisis de calidad temporal CAM

## Objetivo

El analisis de calidad temporal tiene como objetivo revisar la continuidad de las series de tiempo de las estaciones CAM antes de aplicar cualquier metodo de imputacion o interpolacion.

Esta etapa es especialmente importante porque la variable principal del proyecto es `nivel_m`, usada para estudiar y predecir posibles inundaciones. En este contexto, rellenar datos faltantes sin conocer primero la estructura temporal puede alterar los picos de nivel del agua y afectar el analisis.

## Ubicacion del codigo

El codigo se encuentra en:

```text
ETL_CAM/transform/analisis_calidad_temporal.py
```

## Entrada del proceso

El archivo de entrada es:

```text
ETL_CAM/transform/data/cam_transformado.csv
```

Este archivo debe haber sido generado previamente por el transformador.

## Salidas del proceso

El proceso genera dos archivos principales:

```text
ETL_CAM/transform/data/calidad_temporal/resumen_calidad_temporal.csv
ETL_CAM/transform/data/calidad_temporal/huecos_temporales.csv
```

Tambien genera un archivo de log:

```text
ETL_CAM/transform/analisis_calidad_temporal.log
```

## Que revisa este analisis

El script revisa cada estacion de forma independiente. Para cada una calcula:

- Fecha minima disponible.
- Fecha maxima disponible.
- Frecuencia temporal detectada.
- Cantidad de registros reales.
- Cantidad estimada de registros esperados.
- Cantidad estimada de registros faltantes.
- Porcentaje de cobertura temporal.
- Cantidad de huecos detectados.
- Hueco maximo en minutos.
- Disponibilidad de la variable `nivel_m`.
- Porcentaje de datos disponibles para `nivel_m`.

## Archivo resumen_calidad_temporal.csv

Este archivo contiene una fila por estacion. Sus columnas principales son:

```text
estacion
fecha_minima
fecha_maxima
frecuencia_detectada
frecuencia_minutos
registros_reales
registros_esperados
registros_faltantes_estimados
porcentaje_cobertura_temporal
cantidad_huecos_detectados
hueco_maximo_minutos
nivel_m_disponibles
nivel_m_faltantes
nivel_m_disponible_pct
apto_interpolacion_directa
```

## Archivo huecos_temporales.csv

Este archivo contiene el detalle de los saltos o huecos detectados en la serie temporal. Sus columnas principales son:

```text
estacion
inicio_hueco
fin_hueco
duracion_minutos
registros_faltantes_estimados
clasificacion_hueco
```

La clasificacion del hueco se define asi:

```text
corto: hasta 60 minutos
medio: mayor a 60 minutos y hasta 24 horas
largo: mayor a 24 horas
```

## Por que se hace antes de imputar

La imputacion en series de tiempo no debe hacerse directamente despues de limpiar los datos. Primero se debe saber si los faltantes son pequenos, medianos o grandes.

Para la variable `nivel_m`, esto es aun mas importante porque los picos del nivel del agua pueden indicar riesgo de inundacion. Si se rellenan datos usando promedios historicos sin revisar los huecos, se puede suavizar un evento extremo o crear un comportamiento que no ocurrio realmente.

## Relacion con interpolacion e imputacion

Este analisis no imputa datos. Su funcion es preparar la decision.

Con los resultados se puede definir una regla posterior como:

```text
Huecos cortos: interpolacion temporal.
Huecos medianos: evaluar promedio historico por estacion, mes, dia y hora.
Huecos largos: no imputar automaticamente o marcar como baja confianza.
```

Tambien permite evaluar si la idea de usar datos del ano anterior es viable. Por ejemplo, si falta una fecha especifica, podria compararse con el comportamiento historico de la misma estacion en fechas similares. Sin embargo, esto debe hacerse con precaucion porque los niveles de agua dependen de eventos reales de lluvia y condiciones hidrologicas del momento.

## Variable principal

La variable prioritaria del analisis es:

```text
nivel_m
```

Esto significa que aunque el archivo tenga otras variables meteorologicas o hidrologicas, el diagnostico pone especial atencion en la disponibilidad de nivel del agua.

## Comando de ejecucion

Desde la raiz del proyecto:

```powershell
python ETL_CAM\transform\analisis_calidad_temporal.py
```

## Interpretacion recomendada

Si una estacion tiene alta cobertura temporal y huecos cortos, puede ser buena candidata para interpolacion directa.

Si una estacion tiene huecos medianos, se debe revisar si conviene usar promedios historicos, ventanas temporales o informacion de variables auxiliares como lluvia.

Si una estacion tiene huecos largos o bajo porcentaje de `nivel_m`, no se recomienda imputar automaticamente para modelar inundaciones sin una justificacion adicional.
